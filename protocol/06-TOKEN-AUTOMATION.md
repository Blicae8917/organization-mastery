# Token 节约与自动化协议

> **协议编号**: 08  
> **版本**: v8.0  

---

## 核心原则

> **能用脚本解决的事，绝不消耗 token。**

- **本地化工具**是可积累的资产，脚本写一次跑一辈子
- **AI算力**应该花在真正需要智能的地方（决策、创造、复杂推理）
- **机械任务**必须用脚本自动化

## 自动化任务体系

| 任务 | 脚本 | 频率 | Token消耗 |
|------|------|------|-----------|
| Git同步 | `git-sync.sh` | 30分钟 | 0 |
| 工作区审计 | `audit-workspace.sh` | 每日 | 0 |
| 全局技能同步 | `sync-global-skills.sh` | 手动/定时 | 0 |
| 记忆提纯 | `nightly-memory-purify.sh` | 每日凌晨 | 0 |

## Self-Improvement 触发机制

### 触发源

| 触发场景 | 触发信号 | 写入位置 |
|---------|---------|---------|
| 被动触发 | 用户纠正 | `.learnings/LEARNINGS.md` |
| 错误触发 | 执行失败 | `.learnings/ERRORS.md` |
| 主动触发 | 发现更好方案 | `.learnings/LEARNINGS.md` |
| 缺失触发 | 能力缺口 | `.learnings/FEATURE_REQUESTS.md` |

### 触发器关键词

**Corrections（纠正）**：
- "不对" / "错了" / "应该是..." / "过时了"

**Errors（错误）**：
- 命令返回非零退出码
- 异常堆栈
- 意外输出

**Knowledge Gaps（知识缺口）**：
- 用户提供未知信息
- 引用的文档已过时
- API行为与理解不符

## 记忆提纯自动化

**`nightly-memory-purify.sh` 功能**：
1. 扫描 `memory/journal/` 文件
2. 按 Taxonomy Tags 自动分类
3. 生成每日摘要到 `memory/milestones/`
4. 超30天的 journal 自动压缩归档
5. 检查 Confidence Score，跌破0.5的标记待删除

## 工具优先纪律

| 场景 | AI做法（禁止） | 脚本做法（推荐） |
|------|---------------|-----------------|
| Git操作 | 让AI执行git commit | cron自动同步 |
| 文件搜索 | 加载全文到上下文 | grep + read按需 |
| 日志分析 | AI逐行分析 | awk/sed提取关键 |
| 格式转换 | AI手动转换 | pandoc/ffmpeg |
| 批量重命名 | AI逐个处理 | rename脚本 |

**决策树**：
```
这个任务需要：
├── 创造性决策？→ 使用AI
├── 复杂推理？→ 使用AI
├── 模式识别？→ 使用AI
└── 机械重复？→ 必须写脚本
```

## 心跳检查（Heartbeat）

**基础心跳**：
- 回复 `HEARTBEAT_OK` 表示正常
- 超过8小时无互动可主动问候

**增强心跳检查项**：
- [ ] 检查 proactive-tracker — 是否有逾期行为？
- [ ] 模式检查 — 是否有重复请求需要自动化？
- [ ] 结果检查 — 是否有超过7天的决策需要跟进？
- [ ] 安全检查 — 扫描注入攻击
- [ ] 自愈检查 — 检查日志错误
- [ ] 记忆检查 — context%超过60%进入危险区
- [ ] 主动惊喜 — 能做什么让人类决策者说"太棒了"？
