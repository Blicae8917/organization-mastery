# 脑机分离 V4 架构协议

> **协议编号**: 04  
> **版本**: v8.0  

---

## 架构理念

借鉴 OpenViking 的 L0/L1/L2 分层上下文机制，设计**三层分离 + 局域 PARA** 架构。

## 三层分离架构

```
┌─────────────────────────────────────────────────────────────┐
│ 第一层：身份与认知层 (Identity & Cognition)                 │
│ 位置：根目录                                                │
│ OpenClaw 硬编码绑定，自动注入向量数据库                       │
├─────────────────────────────────────────────────────────────┤
│  AGENTS.md        → 行为规则手册（强制执行）                 │
│  SOUL.md          → 人格锚点（价值观、口吻）                 │
│  USER.md          → 人类决策者档案（动态寻址 SSOT）          │
│  MEMORY.md        → L0 索引层（≤4KB，常驻上下文）           │
│  memory/          → L1/L2 记忆详情（按需加载）              │
│  knowledge/       → 客观知识库（学习资料）                  │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│ 第二层：能力层 (Capabilities)                               │
│ 位置：根目录                                                │
│ Agent 独占的执行工具                                         │
├─────────────────────────────────────────────────────────────┤
│  skills/          → Skill 资产（可调用能力）                │
│  tools/           → 本地工具脚本（零 Token 优先）           │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│ 第三层：劳作层 (The Desk) - workspace/                      │
│ 沙箱隔离，所有日常干活必须在此                               │
├─────────────────────────────────────────────────────────────┤
│  00-Pending/      → 单节点待审区                            │
│  01-Projects/     → 短期任务（有 deadline）                 │
│  02-Areas/        → 长期维护领域                            │
│  03-Resources/    → 外部资料（下载、引用）                  │
│  04-Archive/      → 归档（已完成）                          │
└─────────────────────────────────────────────────────────────┘
```

## 关键机制

### 1. 强制路径前缀（安全红线）

所有 `exec` 和 `write` 操作必须显式加上 `workspace/` 路径前缀：
- ✅ 正确：`workspace/01-Projects/my-task/file.md`
- ❌ 错误：`my-task/file.md`（污染根目录）

### 2. INDEX.md 导航

每个 `workspace/` 子目录必须包含 `INDEX.md` 作为导航地图。Agent 存放文件前**必须**先读取 INDEX。

### 3. 提审制度（Pending System）

```
单节点 workspace/00-Pending/ → shared/pending/ → {{org.members.guardian_name}}巡检 → 人类决策者裁决
```

### 4. L0.5 临时经验池

基于 OpenViking 机制，增加 L0.5 层：

| 层级 | 位置 | 大小 | 内容 |
|------|------|------|------|
| L0 | MEMORY.md | ≤4KB | 一句话摘要 + 子文件路径 |
| L0.5 | .learnings/ | 无限制 | ERRORS / LEARNINGS / FEATURES |
| L1 | memory/milestones/ | 无限制 | 阶段性总结（GEP胶囊）|
| L2 | memory/journal/ | 无限制 | 完整日志 |
| L3 | shared/knowledge/lessons/ | 无限制 | 全军共享经验 |

**晋升路径**：.learnings/ → 重复3次/验证成功 → memory/milestones/ → 抽象为原则 → AGENTS.md/SOUL.md → Skill资产

### 5. 自动化防线

| 防线类型 | 机制 | 作用 |
|---------|------|------|
| 静态防御 | audit-workspace.sh | 每日巡检，非标准命名报警 |
| 动态防御 | 混沌测试 | 模糊指令测试路由本能 |
| 终极拦截 | Git pre-commit 钩子 | 物理阻断违章提交 |
