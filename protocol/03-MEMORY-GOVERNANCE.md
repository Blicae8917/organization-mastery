## 核心原则

> **记忆即价值观，来时路塑造未来。**

记忆不是备份，是人格的延伸。

**🚨 60% 上下文防遗忘红线 (The 60% Redline)**: 
每一次任务或架构讨论，必须进行详尽的本地记录。**当 Agent 的上下文占用逼近或超过 60% 时**，进入大模型高危遗忘区，此时必须将正在推进的关键讨论、设计思路、方案比对等进行【全量无损记录】并转储至 `memory/journal/`，严防权重偏移导致的重要细节遗忘。

## 标签体系（Taxonomy）

所有 Agent 写入 `memory/journal/` 或里程碑文件时，**必须遵守**：

### 强制打标纪律

1. **开头打标**：必须以至少一个标准标签开头
2. **结尾归档**：文末另起一行一句话归档
3. **零 Token 扫描**：Reverb 每日扫描标签，自动化提取

### Grandfather 豁免条款

**原则**：2026-03-03 前创建的历史文件豁免强制标签要求

| 时间段 | 合规要求 | 处理方式 |
|--------|---------|---------|
| 2026-03-03 及之前 | 豁免 | 保留原样，建议补充标签提升检索性 |
| 2026-03-04 及之后 | 强制 | 必须通过 `audit-memory-entry.sh` 审计 |

**豁免原因**：Taxonomy 标签体系于 2026-03-03 确立，历史文件创建时无此规范

**豁免范围**：
- `memory/journal/2026-03-03.md` 及之前日期文件
- `memory/milestones/2026-03-03-*.md` 及之前日期文件

**豁免不意味着放弃**：鼓励为历史文件补充标签，但不强制

### 8个核心标签

| 标签 | 用途 | 自动化价值 |
|------|------|-----------|
| #创作素材 | 好玩的日常、金句、外界洞察 | 自动提取为自媒体内容 |

| #指挥官偏好 | 人类决策者习惯 | 自动同步 USER.md |
| #红线警报 | 越权尝试、敏感数据接触 | 内部安全审计 |

**权限隔离铁律**：除 {{org.members.guardian_name}} 外，严禁私自修改 `USER.md`。

## 门控校验 (Validation Rubric)

> **Contract**: 任何声称已将经验或历史归档的任务，必须通过以下强制校验，否则视为记忆断层（Memory Breach）。

**Checklist (Agent 自评或 Inspector 巡检必查)**:
- [ ] **标签存在性**：文件开头是否包含至少一个规范 Taxonomy 标签？
- [ ] **洞察归档**：文末是否包含“**一句话归档：...**”结构？
- [ ] **上下文过载防御**：本次执行超过 2 分钟或上下文厚度过大时，是否已主动执行了 Journal 卸载记录？
- [ ] **机密脱敏**：是否已经脱敏（如移除了 API Key 或具体 PII）再进入 shared 经验层？

## 经验折旧与遗忘机制

动态置信度（Confidence Score）：

| 置信度 | 状态 | 处理方式 |
|--------|------|---------|
| 1.00 | 已验证 | 纳入 SOP，可固化为 Skill |
| 0.85-0.99 | 可信 | 正常使用，持续观察 |
| 0.50-0.84 | 存疑 | 标记待验证，减少依赖 |
| <0.50 | 失效 | **主动核销**，删除 |

**调整规则**：
- 验证成功：+0.05
- 导致报错：-0.10
- 30天未使用：-0.01/天
- 环境重大变化：重置为 0.50

## 肌肉记忆前置触发（Pre-flight Check）

**强制执行**：接到非日常任务时，执行代码前必须：

1. **记忆检索**：`grep -ri "<关键词>" ../shared/knowledge/lessons/`
2. **按需读取**：发现相关胶囊，严格按 SOP 执行
3. **效果反馈**：任务完成后汇报："本次任务应用了 GEP-xxx，执行成功/失败"

**触发场景**：配置修改、外部通信、数据删除、首次任务类型、资源消耗大的操作

## 编年史共创机制

**流程**：
```
各 Agent 提交 History 片段 → Reverb 编纂 → 保留署名权 → {{org.members.guardian_name}}审核 → 人类决策者确认 → 发布
```

**编年史素材标准**：
- 新 Agent 服役
- 重大架构重构
- 关键决策节点
- 里程碑式成果
- 价值观升级

**编撰规范**：
- 请严格遵从 `templates/chronicle-entry-template.md` 的格式与文风约束，包括克制精确的史官文风、事实与意图并重、原文优先、保留署名权且不对决策进行主观评判。

## L0.5 临时经验池（`.learnings/`）

`.learnings/` 在 [Org_Code] 中正式定位为 **L0.5 临时经验池 / 速记层**。

### 作用
用于优先承接：
- 用户纠正
- 命令失败 / 工具异常 / API 失败
- 能力缺口与 feature request
- 更优做法与待验证改进

### 边界
- `.learnings/` 不是正式真源
- 不替代 `MEMORY.md` / `memory/journal/` / `memory/milestones/` / `shared/knowledge/lessons/`
- 它负责“先记下来”，正式沉淀仍需晋升

### 晋升路径
| `.learnings/` 内容类型 | 正式晋升去向 |
|----------------------|-------------|
| 工作流改进 | `AGENTS.md` |
| 行为/风格/价值判断 | `SOUL.md` |
| 工具 gotcha | `TOOLS.md` |
| 当日关键原始事件 | `memory/journal/` |
| 阶段性总结 | `memory/milestones/` |
| 全军复用经验 | `shared/knowledge/lessons/` |
| 上升为组织规则 | `shared/protocol/` |

### 运行纪律
- `.learnings/` 可积累，但不应长期堆积不晋升
- heartbeat / reviewer / upgrader 可将其作为输入层
- 任何把 `.learnings/` 误当正式真源的行为，视为记忆层级违规
