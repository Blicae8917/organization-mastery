# 战术面板协议（MISSION_CONTROL）

> **协议编号**: 07  
> **版本**: v8.0  

---

## 核心定位

MISSION_CONTROL.md 是**跨节点协同的实时任务看板**，所有 Agent 共享同一视图。

## 文件结构（索引层）

```markdown
# MISSION_CONTROL.md

## 当前活跃任务
| 任务ID | 状态 | 负责人 | 优先级 | 截止日期 |
|--------|------|--------|--------|---------|
| TASK-001 | 🟡 进行中 | Echo | P0 | 2026-03-21 |

## 紧急标记
🔴 TASK-003: 需立即处理

## 详情
完整协议见 protocol/07-MISSION-CONTROL.md
```

## 任务状态定义

| 状态 | 标识 | 说明 |
|------|------|------|
| 🔴 阻塞 | blocked | 需外部输入，无法继续 |
| 🟡 进行中 | in_progress | 正在执行 |
| 🟢 待启动 | pending | 已分配，未开始 |
| ✅ 已完成 | completed | 已交付 |
| ⏸️ 暂停 | paused | 主动暂停，后续恢复 |
| 🗑️ 已取消 | cancelled | 不再执行 |

## 优先级定义

| 优先级 | 含义 | 响应时间 |
|--------|------|---------|
| P0 | 紧急 | 立即 |
| P1 | 重要 | 当日 |
| P2 | 一般 | 本周 |
| P3 | 低优先级 | 按需 |

## 任务ID规范

格式：`{项目代号}-{任务类型}-{序号}`

示例：
- `AI-2026-005-001`：AI项目2026年第5个任务的第1个子任务
- `ARCH-V8-001`：架构版本8的第1个任务
- `OPS-DAILY-001`：日常运维任务

## 跨节点任务协调

### Ark 分发机制

1. 人类决策者或 {{org.members.guardian_name}} 创建任务
2. Ark 评估任务类型和所需能力
3. Ark 唤醒对应 Agent（或分配给现有活跃 Agent）
4. 更新 MISSION_CONTROL.md
5. Agent 执行完成后汇报给 Ark
6. Ark 更新状态，触发下一步或归档

### 信息传递规范

- **任务分配**：Ark → Agent（通过 `sessions_spawn` 或 `sessions_send`）
- **进度汇报**：Agent → Ark
- **紧急升级**：Agent → {{org.members.guardian_name}} → 人类决策者
- **结果归档**：Ark → Reverb（编年史记录）

## 紧急标记处理

🔴 标记触发条件：
- 安全事件
- 系统故障
- 超出预设阈值的资源消耗
- 跨节点通信失败
- 人类决策者直接标记

处理流程：
1. 发现紧急标记的 Agent 立即暂停其他任务
2. 评估是否需要熔断（暂停相关 Agent）
3. 通知 {{org.members.guardian_name}} 和 人类决策者
4. 等待决策后执行

## 定期同步机制

- **实时**：关键状态变更立即同步
- **定时**：Ark 每 30 分钟检查一次任务状态
- **心跳**：各 Agent 响应心跳时汇报当前任务进度
