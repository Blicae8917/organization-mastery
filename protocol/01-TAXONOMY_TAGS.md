# 8917 全局标签库 (Taxonomy Tags)

> 本文件定义了 8917 数字集团军在执行任务、编写日志 (journal) 和复盘时必须使用的标准标签体系。
> 这是编年史官 (Reverb) 进行零 Token 自动化数据提炼的底层依赖。

## 一、打标操作规范 (强制纪律)

所有 Agent 在执行任务后写入 `memory/journal/` 或里程碑文件时，**必须遵守** 以下格式：
1. **开头打标**：必须以至少一个标准标签开头。
2. **结尾归档**：必须在文末另起一行写入：“**一句话归档：<用 8917 视角提炼的核心洞察/变更点>**”。
3. **权限隔离铁律**：除参谋长（Echo）外，全军所有 Agent **严禁**私自修改 `USER.md` 或其他全局配置文件。发现环境或指挥官偏好变化，只能打 `#指挥官偏好` 或 `#技术踩坑` 标签，由编年史官扫描后提审。

## 二、核心标签定义 (8 个)

### 【对外输出/灵感类】
*   `#创作素材`：好玩的日常、金句、外界洞察，或值得对外输出为自媒体内容的“弹药”。

### 【技术与架构类】
*   `#技术踩坑`：报错排查、网络阻断、API 限流及解决思路（避免二次踩坑）。
*   `#决策节点`：重要的架构或策略选择，记录“选了 A 没选 B 的原因”。
*   `#SOP沉淀`：跑通的标准化工作流（未来可固化为 Skill 或独立 Agent）。

### 【组织与灵魂类】
*   `#价值观升级`：8917 哲学的演进（如 D.C.E. 协议、少帅视图、Token 节约主义）。
*   `#编年史素材`：值得载入 8917 史册的里程碑（新 Agent 服役、重大重构、重大阶段性成果）。
*   `#指挥官偏好`：记录指挥官（叶澄风）的特定习惯、禁忌或身份设定。扫描后将由参谋长更新到全局 `USER.md`。
*   `#红线警报`：越权尝试、敏感数据接触、触发 D.C.E 熔断的记录（专供内部安全审计）。
